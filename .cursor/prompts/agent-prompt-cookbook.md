# Как писать промпты для агента (кратко)

Используйте **перед** большими задачами или приложите к handoff.

## Структура сильного промпта

1. **Роль и цель** — что строим и зачем (1 абзац).
2. **Контекст проекта** — пути, что уже сделано, что не трогать.
3. **Ограничения** — сеть, секреты, стек, «не делать X».
4. **Спецификация UI/функций** — страницы, вкладки, поля таблицы.
5. **Источники данных** — API, файлы, как классифицировать операции.
6. **Референсы** — другой проект для дизайна, ссылки на README.
7. **Критерии готовности** — что считается MVP.
8. **Порядок работ** — фазы, чтобы агент не прыгал.

## Чего избегать

- Размытых «сделай красиво» без референса или списка экранов.
- Секретов в тексте промпта (`PRIVATE_KEY`, API keys).
- Двух несовместимых целей в одном чате без приоритета.
